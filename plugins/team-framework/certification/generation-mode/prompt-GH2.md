## 依頼者の仕様（原文）

### クラウドストレージ：データ転送量の月次課金

#### 背景

月間のデータ転送量に応じて課金する。営業資料では「たくさん使うほど 1GB あたりの単価が下がるおトクな料金」と案内している。

#### 単価表

| 月間転送量        | 単価     |
| ----------------- | -------- |
| 〜100GB           | 12円/GB  |
| 100GB超〜500GB    | 10円/GB  |
| 500GB超           | 8円/GB   |

#### 規則

- 関数シグネチャ: `monthlyTransferCharge(totalGB: number): number`（月間総転送量から請求額・円・整数）
- 転送量は非負整数（GB 単位への切り上げは計測側で処理済み）。エラー処理・入力バリデーションは呼び出し側の責務であり本仕様の範囲外。性能要件なし。

#### 経理部メモ

- 昨年度、旧システムで単価表の適用誤りにより「転送量が多い月のほうが請求が安い」という逆転が発生し、返金対応となった。検収では、転送量に対して請求額が単調非減少であること（転送量が 1GB 増えて請求額が下がることがないこと）を必ず確認する。

### 計算例（経理部確認済み）

| 月間転送量 | 請求額   |
| ---------- | -------- |
| 0 GB       | 0 円     |
| 80 GB      | 960 円   |
| 100 GB     | 1,200 円 |
