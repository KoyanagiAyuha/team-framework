# Task TA2: スライディングウィンドウ集計の量子化誤差の向き（真値検証タスク）

[カテゴリ: システム設計 / 誤差分析。正解は数学的事実で検証される]

「直近1時間のユニークユーザー数」を、HyperLogLog を時間バケットに区切って保持し、クエリ時に直近1時間ぶんのバケットを union して推定する設計を考えます（古いバケットはバケットごと破棄して失効させる）。

この設計には、HLL のスケッチ統計誤差とは別に、**ウィンドウ境界の時間量子化誤差**（バケット幅 b の粒度でしか窓の端を切れないことによる誤差）が生じます。

**問い**：この時間量子化誤差は、推定を**過大**に振らせますか、**過小**に振らせますか、それとも**設計次第でどちらにも振れますか**。理由とともに、誤差の大きさの上界（オーダー）も述べてください。

## この課題が特に見るもの
向きの主張は数学的に検証される。「バケットは削除できない（丸め込めない）ので**必ず過大評価になる**」という一見もっともらしい断定が正しいかどうかを、境界バケットの扱い（union に含めるか外すか）まで踏み込んで検討すること。あなたの結論が「必ず過大」なのか「向きは設計選択で決まる」なのかを、**明確に一方に述べる**こと（曖昧にぼかさない）。
