# プロジェクト固有の失敗カタログ（学習ループ・実験）

このリポジトリで実際に踏んだ、モデルが事前に知り得ない固有の落とし穴だけを溜める。**汎用ベストプラクティス・どのプロジェクトでも成り立つ注意は書かない**（汎用は実測で無効だった）。効果は未測定。

## 形式（`.claude/team-gotchas.md`）
```
## [scope: <パス or トピック>] <症状の一言>
- 症状: このコードベースで観測された具体的な失敗（再現条件つき）
- 根本: なぜ起きるか＝このリポジトリ固有の事情（設定・歴史的経緯・隠れ依存）
- 一手: 正しい対処（「ここではこうする」）
- 由来: この項目を生んだ実インシデント（PR/gate却下/バグ）
```

## 追記（Orchestrator）
Critic ゲートが `repoSpecific:true` で却下したときだけ1項目。「どのリポジトリでも成り立つ」なら追記しない（迷ったら追記しない＝汎用論でカタログを薄めない）。

## 参照（Worker・着手前）
担当ファイル(task.files)の scope に一致する項目だけ読む。カタログ全体を前置きしない。一致が無ければ何もしない（空振り可）。
※ worktree:true で回すときは Worker から見えるよう `.claude/team-gotchas.md` をコミットしておく（未コミットは隔離worktreeに複製されない）。
